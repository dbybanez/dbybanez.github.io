How it works


So, let's proceed on how we generated the string send nudes.

This expression [![]] returns false while this expression [][[]] returns undefined. When both expressions are concatinated using +, it returns a string falseundefined. In order to get each letter in the string, just access it like how you normally access an array. An example of this is arr[] but instead of using a variable named arr, we only used expressions.

So if we want to access letter "s" in the string, just do this expression ([![]]+[][[]])[(!+[]+!+[])+(++[[]][+[]])]. Confusing right? Don't give up just yet!

Let's try to understand this, ([![]]+[][[]])[(!+[]+!+[])+(++[[]][+[]])]. This expression ([![]]+[][[]]) returns the string falseundefined, right? Notice the following expressions looks like it's enclosed in [] which denotes an array. This means that the expression inside [] must be an integer. Inside the [], you can see that 2 expressions were added. The first operand (!+[]+!+[]) returns a value of 2 while the second operand (++[[]][+[]]) returns a value of 1. In short, our expression inside [] looks like this 2 + 1. To further explain why we got a value of 1, notice that at the beginning of the second operand, there's an expression ++. The expression ++ basically means incrementing a value. Since we've added it at the beginning of an expression, the value is incremented before it is returned. In programming, this is what we call pre-increment. The expression [+[]] returns the index of 0. Notice how it is enclosed with [], this means that the expression ++[[]][+[]] is an array.

When both operands are added, it returns a value of 3. So when we convert it to a simpler expression using a variable, it looks like this arr[3] where arr is an array of characters while 3 is the number we used to access the index of an array.
